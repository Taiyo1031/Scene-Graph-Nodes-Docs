## Rules

- Keep English and Japanese pages in sync.
- Add one page per node.
- Keep slugs stable after publication.
- Update `data/nodes.json` whenever node classes are added, renamed, or removed.
- Prefer short examples that map to actual node chains.

## Page Template

Every node page must include purpose, inputs, outputs, properties, evaluation behavior, example, limitations, `node_id`, `node_class`, and `added` front matter.
